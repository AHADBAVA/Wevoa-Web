# WevoaWeb Roadmap

This roadmap reflects the current direction of the project and may evolve as the language matures.

## Near Term

- Improve test coverage across the lexer, parser, interpreter, and server
- Load `wevoa.config.json` automatically in the CLI and dev server
- Improve error reporting and diagnostics
- Expand the starter project templates
- Add more examples and contributor docs

## Language Goals

- Arrays and map-like data structures
- More operators and control flow
- Module and import system
- Better standard library primitives
- Stronger template and string ergonomics

## Runtime Goals

- Bytecode VM
- Performance profiling
- Better native extension points
- Cleaner separation between language runtime and web framework runtime

## Web Framework Goals

- Layout and template composition
- Route parameters
- POST request handling
- Better static file support
- Config-driven server settings

## Tooling Goals

- Production build workflow
- Packaging and release process
- Better project creation templates
- More polished logging and diagnostics
