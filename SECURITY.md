# Security Policy

## Supported Status

WevoaWeb is currently experimental and moving quickly.

Supported security fixes are best-effort for:

- the latest `main` branch state
- the latest tagged public release

Older versions may not receive fixes.

## Reporting a Vulnerability

Do **not** open public issues for security-sensitive problems.

Use one of these paths instead:

- GitHub private vulnerability reporting, if enabled
- the repository owner's private security contact

If you are the repository owner, add your preferred security contact details on GitHub and in this file before a wider public launch.

Instead, report:

- the affected component
- the impact
- steps to reproduce
- the smallest proof-of-concept needed to explain the issue
- any suggested fix if available

## Response Goals

The project will try to:

- acknowledge reports promptly
- confirm reproducibility
- prepare a fix or mitigation
- document any important user-facing guidance

## Disclosure Expectations

Please give maintainers reasonable time to investigate and prepare a fix before public disclosure.

For issues that could expose users to immediate harm, maintainers may:

- prepare a private fix first
- publish a patch release
- publish limited mitigation guidance before full technical detail
